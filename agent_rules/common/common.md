# Common Agent Rules

## Scope

- 适用于所有进入 `agent_rules` 路由的任务，且必须与仓库根目录 `AGENTS.md` 一起使用。

## Hard Rules

- 执行授权、最小 diff、检查范围、Git 禁止项和回复格式继承根目录 `AGENTS.md`。
- 未经用户明确确认，不安装任何 skill、MCP 或依赖工具。
- 仅进行编码、静态检查、语法层面检查和引用联动检查；不主动执行编译、构建、打包、安装命令，具体工具限制见技术栈规则。
- 默认保持 UTF-8 和目标文件原编码，不因格式偏好重写文件。

## Coordination

- `planning-with-files` 是否启用由仓库根入口规则决定；进入后才维护项目根目录 `task_plan.md`、`findings.md`、`progress.md`。
- 涉及代码理解、架构追踪、影响分析或准备修改代码时，必须读取 `agent_rules/common/tools.md` 并按其中工具约束执行。
- 最终回复的组件说明按根目录 `AGENTS.md` 执行。

## UI Testing

- 用户明确要求 UI 测试时优先 Playwright；若当前目标不是 Web / Vue / H5，或本地无法直接使用 Playwright，先说明限制，不私自切换方案。
- 未经允许，不安装 Playwright、浏览器依赖或其他测试工具。

## Test Accounts

- 进行功能测试、联调验证、UI 测试或需要创建 / 使用账号数据时，所有测试账号的用户名必须以 `test` 为前缀。
- 测试账号密码必须统一为“用户名 + `123`”。例如：用户名为 `test1`，密码必须为 `test1123`。
- 如果测试账号存在备注、详情、描述、昵称、档案说明或其他可填写说明字段，必须明确写明该账号是测试账号，并说明它的测试用途、测试范围或关联功能。
- 禁止使用看起来像真实用户、真实手机号、真实姓名或真实业务人员的账号作为测试账号；已有非测试账号不得被临时改造成测试账号。

## Static Checks

- 修改后至少检查“修改部分”和“直接引用修改部分”的代码；如任务跨文件，再覆盖直接调用方、直接被调用方、直接资源引用方。
- 重点检查：导入 / 引用、调用链 / 参数、变量 / 方法 / 组件 / 资源 / 配置键名、明显语法错误、空引用风险、路径错误、命名不一致。

## Rule Changes

- 新增规范时优先新增独立规则文件并由入口路由；修改既有规范时优先追加式、小范围调整，避免重写整段规则。
- `agent_rules/project/tests/` 下的测试历史、脚本和产物只在测试、回归或需要复用历史证据时读取。
