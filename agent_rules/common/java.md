# Java Agent Rules

## Scope

- 任务命中 Java 标记时，除仓库根目录 `AGENTS.md` 和 `agent_rules/common/common.md` 外，还必须读取本文件。

## Edit Rules

- 仅进行编码、静态检查、语法检查和引用联动检查；Maven 相关命令必须按 `agent_rules/project/project.md` 中的 Maven 执行路径授权规则执行。
- 保持现有模块边界、包结构、配置方式，不做无关重构。
- 修改类、接口、枚举、注解或配置后，检查直接引用方；修改资源或配置文件后，检查 `pom.xml`、配置读取方、入口类、框架装配代码的直接引用。

## Database Schema Changes

- 任务涉及数据库表结构变更时，必须同步更新数据库初始化 SQL，使空库初始化结果就是最新结构。
- 表结构变更任务的执行顺序固定为：先完成代码与初始化 SQL 修改，再在相应 SQL 目录生成唯一一个可直接执行的增量 SQL。
- 增量 SQL 必须自包含、可重复识别执行前置条件，并覆盖字段新增、索引、约束、数据回填和必要校验；禁止把同一任务拆成多个互相依赖的增量 SQL。
- 只生成增量 SQL，不代替用户执行；用户确认并自行执行增量 SQL 后，再进入数据库结果验证或后续联调。

## Static Checks

- 检查 `import`、包名、类名、方法签名、构造参数、泛型参数、返回值是否与调用方一致。
- 检查注解、配置键、资源路径、接口实现类、工厂、注册表、反射入口是否仍然闭合。
- 若项目同时命中 Android 标记，则按 Android 规则优先处理。
